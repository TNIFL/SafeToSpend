MVP
계좌를 연결하면 거래내역이 자동으로 들어오고, 
매출/경비가 자동 분류되며, 
‘이번 달 따로 빼둘 세금’과 ‘누락 증빙’이 한 화면에 뜬다.

여기서 세금 신고 대행 / 홈택스 자동제출 / 절세 컨설팅은 MVP 아님.
MVP는 “정리 자동화 + 불안 제거(얼마 빼야 하지?)”가 전부

꼭 되어야하는 기능
1. 계좌 연결 -> 거래 자동 수집 (팝빌)
    
2. 자동 분류 + 사용자가 30초 안에 수정 가능
    기본 분류: 매출(입금) / 경비(출금) / 이체 / 보류

3. 월간 보드 1개
    이번 달 매출 합계
    이번 달 경비 합계
    이번 달 따로 빼둘 세금(추정치) 
        (정확한 신고액이 아니라 “적립 권고액”)
    증빙 누락 경고(아직 안 올린 항목 수)

다운로드 되는 엑셀 파일 디자인, 컬럼 명 등등 수정할거 많음.


1. 영수증 비용처리 했을 때 세금에서 어느정도가 면제가 되는지, 면제가 된다면 바로 세금 얼마나 낼지 반영하기
2. 영수증을 사진으로 찍어서 업로드 하면 그걸 읽어와서 파싱 후 데이터로 저장시키는 방식 있으면 좋을거같아. LLM 사용하거나 제공되는 API 사용해서 만들어야함
3. 다운로드 되는 엑셀 파일 디자인, 컬럼 명, 등등 수정할거 아직 있음
4. 사용자가 업로드 하는 이미지를 업로드 당시 파일 명 그대로 올리지 말고, 예를들어 오픈뱅킹 출금 이면, 증빙자료 업로드 하고나서, 자동으로 파일 명이 사용자이름_은행명_계좌번호_증빙자료(1) 이런식으로 업로드 되도록
5. 세무사에게 파일 전달할 때 가족관계증명서, 사업자등록증 사본, 신분증 사본, 홈택스 아이디/비밀번호 등 이걸 내 서비스에 저장해서 세무사에게 전달하는 패키지에 넣는게 옳은 판단인지.
6. 이건 추가로 생각해볼건데, 차라리 세무사들도 내 서비스에서 세무 관련 업무를 할 수 있게 하는건 어떤지?


회귀 체크리스트(세금/건보료 반영)
1. 원천징수 없음/있음 비교
- 내 정보에서 원천징수(3.3%)를 없음으로 저장 후 `/dashboard/tax-buffer`의 추가 납부 예상세액을 기록한다.
- 원천징수를 있음으로 바꾼 뒤 새로고침해서 추가 납부 예상세액이 감소하는지 확인한다.

2. 건보료 월 납부액 반영
- 내 정보에서 건강보험을 지역 + 월 납부액(예: 100000)으로 저장한다.
- `/dashboard/review`와 `/dashboard/tax-buffer`에서 "건보료 보관(이번 달)"이 같은 금액으로 보이는지 확인한다.
- "이번 달 총 보관 권장액 = 세금 보관 권장 + 건보료 보관" 합계가 맞는지 확인한다.

3. 프로필 미완료/기본값 처리
- 업종/전년도 수입/원천징수를 모름으로 두고 페이지를 새로고침한다.
- 두 화면에서 "기본값 추정" 배지와 안내 문구가 보이는지 확인한다.
- "내 정보 수정" 링크가 동작해 `/dashboard/profile`로 이동하는지 확인한다.
