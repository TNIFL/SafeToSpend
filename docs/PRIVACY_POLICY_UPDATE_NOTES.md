# 개인정보처리방침 업데이트 노트

## 이번 업데이트가 필요한 이유
- 공식 자료 업로드/분석 기능을 도입하려면 일반 계정 정보보다 더 구체적인 수집·이용 목적, 항목, 보유기간, 파기 기준을 밝혀야 한다.
- 특히 사용자 업로드 자료, 구조 검증 완료 자료, 기관 확인 완료 자료를 같은 수준으로 설명하면 안 된다.

## 추가 또는 보강이 필요한 항목
### 수집·이용 목적
- 세금/건보료/증빙 정확도 보정
- 세무사 전달 보조
- 기준일이 있는 공식 자료 스냅샷 관리

### 자료 종류
- 홈택스 지출증빙/사업용카드/전자세금계산서/원천징수·지급명세·납부내역
- NHIS 보험료 납부확인서/자격득실확인서

### 기본 저장 단위
- 핵심 추출값 우선 저장
- 원본 파일은 기본 비저장 또는 선택 저장 구조 검토
- 문서 메타, 핵심 추출값, 파생 상태값을 파일 전체 저장과 분리

### 저장하지 않는 항목
- 주민등록번호 전체
- 건강 상세정보
- 서비스 목적과 무관한 긴 문서 원문
- 구조 검증 실패 파일의 긴 preview 텍스트

### 보유기간
- 목적 달성 또는 보유기간 종료 시 지체 없이 파기
- 더 최신 기준 자료로 교체되면 이전 자료를 정리하는 정책 명시
- 업로드 파일 전체는 기본 비저장이고, 선택 저장을 열 경우 별도 보유기간을 분리

### 파기 기준
- 계정 탈퇴
- 목적 종료
- 기준일 경과 및 대체 자료 확보
- 원본 선택 저장 기간 만료

### 안전조치 개요
- 접근 통제
- 전송·저장 구간 보호
- 로그 및 권한 관리
- 민감/고유식별정보 최소화
- 계좌번호, 고유식별정보 가능 값, 기관 확인 메타는 일반 로그/디버그 출력과 분리

### 거부 가능 범위
- 공식 자료 업로드는 선택 사항으로 두고, 거부 시 정확도 보정 기능이 제한될 수 있음을 고지

### 검증 수준 안내
- `기관 확인 완료 여부에 따라 검증 수준이 다름`
- `구조 검증 완료는 기관 진위확인과 동일하지 않음`
- `사용자 업로드 자료 기준으로만 반영될 수 있음`
