# 영수증 비용처리 안내 정책

## A. 문제 요약
- 영수증 자동 분류나 추천이 바로 "비용처리 확정"처럼 읽히면 사용자가 필요경비 인정 여부를 과신할 수 있다.
- 서비스는 규칙 엔진을 만들기 전에 먼저 "어디까지 자동 후보로 볼지 / 어디서 추가 확인으로 내려야 할지"를 보수적으로 고정해야 한다.
- 이번 정책은 비용처리 엔진 자체가 아니라 사용자 안내 UX의 기준 문서다.

## B. 공식 근거 기반 안내 범위

### 4단계 안내 분류

| 단계 | 사용자용 라벨 | 내부 판단 목적 | 공식 근거 출처 | 서비스 문구 톤 | 자동화 수준 |
| --- | --- | --- | --- | --- | --- |
| 1 | 비용처리 가능성이 높은 항목 | 업무 관련성이 비교적 설명되기 쉬운 영수증을 우선 후보로 안내 | 소득세법 제27조, 제160조의2 | 가능성 중심, 단정 금지 | 반자동 후보 |
| 2 | 추가 확인이 필요한 항목 | 개인 사용과 업무 사용이 섞이기 쉬운 지출을 보수적으로 안내 | 소득세법 제27조, 제35조 | 이유 설명 + 추가 메모 유도 | 반자동 + 사용자 확인 |
| 3 | 자동 인정하지 않는 항목 | 가사 관련/개인소비 성격이 강한 지출을 자동 인정에서 제외 | 소득세법 제33조 | 보수적 차단, 자동 확정 금지 | 수동 검토 전제 |
| 4 | 세무 검토 권장 항목 | 자산성 지출·특수 지출처럼 별도 판단이 필요한 항목을 분리 | 소득세법 제27조, 제35조 | 자동 비용처리 대신 검토 권장 | 수동 검토 권장 |

## C. 분류별 포함 항목

### 1) 비용처리 가능성이 높은 항목
| 항목 | 안내 기준 | 자동/반자동/수동 |
| --- | --- | --- |
| 교통비, 주차비, 통행료 | 출장·거래처 방문 등 업무 이동과 직접 연결되는 경우가 많음 | 반자동 후보 |
| 도서, 교재, 교육, 세미나 | 업무 수행과 직접 관련된 학습·자료 취득으로 설명되기 쉬움 | 반자동 후보 |
| 인쇄, 복사, 우편, 택배 | 업무 산출물·발송과 연결되기 쉬움 | 반자동 후보 |
| 사무용 소모품 | 문구, 복사용지, 토너 등 소모품 성격이 비교적 명확함 | 반자동 후보 |

### 2) 추가 확인이 필요한 항목
| 항목 | 안내 기준 | 자동/반자동/수동 |
| --- | --- | --- |
| 거래처 식사/접대비 후보 | 접대 목적, 참석자, 적격증빙 여부를 함께 확인해야 함 | 반자동 + 사용자 확인 |
| 카페, 식비, 음료 | 영수증만으로 업무 미팅인지 개인 소비인지 단정하기 어려움 | 반자동 + 사용자 확인 |
| 주말/심야 결제 | 시간대만으로 업무 관련성을 설명하기 어려워 추가 맥락 필요 | 반자동 + 사용자 확인 |
| 개인/업무 혼합 가능 지출 | 편의점, 온라인 쇼핑 등 혼합 사용 가능성이 높음 | 반자동 + 사용자 확인 |

### 3) 자동 인정하지 않는 항목
| 항목 | 안내 기준 | 자동/반자동/수동 |
| --- | --- | --- |
| 본인 식비·음료 | 가사 관련 소비일 가능성이 높아 자동 인정 금지 | 수동 검토 전제 |
| 생활비·가정용 소비 | 가사 관련 경비는 필요경비에 산입되지 않을 수 있음 | 수동 검토 전제 |
| 개인 쇼핑·선물성 지출 | 업무 관련성이 별도로 설명되지 않으면 자동 인정 금지 | 수동 검토 전제 |

### 4) 세무 검토 권장 항목
| 항목 | 안내 기준 | 자동/반자동/수동 |
| --- | --- | --- |
| 고가 전자기기 | 자산 처리·감가상각 여부 검토 필요 가능성 | 세무 검토 권장 |
| 고가 가구·장비 | 단순 소모품보다 자산 취득으로 볼 여지가 있음 | 세무 검토 권장 |
| 경조사비 관련 증빙 | 일반 경비와 다른 제한 규정·사실관계 확인 필요 | 세무 검토 권장 |

## D. 사용자 안내 문구 원칙
- 허용 표현
  - 비용처리 가능성이 높은 편이에요
  - 추가 확인이 필요해요
  - 자동으로 인정하지 않아요
  - 세무 검토가 필요할 수 있어요
- 금지 표현
  - 무조건 비용처리 됩니다
  - 100% 인정됩니다
  - 세무서에서도 반드시 인정됩니다

## E. 인라인 설명 반영 결과
- review/receipt 화면에서는 긴 법률 설명 대신 1~2줄 요약을 우선 노출한다.
- 모든 인라인 설명은 아래 구조를 따른다.
  - 상태 라벨 1개
  - 짧은 사유 1줄
  - `왜 이렇게 보나요?` 링크 1개
- `왜 이렇게 보나요?` 링크는 `/guide/expense`의 해당 섹션 anchor로 연결한다.

## F. 고지 문구 반영 결과
- 서비스의 분류 결과는 보조 판단입니다.
- 최종 필요경비 인정 여부는 실제 거래 사실, 증빙, 법령, 과세관청 판단에 따라 달라질 수 있습니다.
- 개인지출 또는 가사 관련 경비는 필요경비에 산입되지 않을 수 있습니다.

## G. 남은 리스크
- 교통비·도서·소모품처럼 일반적으로 설명이 쉬운 항목도 실제 업무 관련성 자료가 없으면 분쟁 가능성이 남는다.
- 거래처 식사·접대비·경조사비는 영수증만으로 판단하기 어려워, 향후 규칙 엔진에서도 추가 질문 구조가 필요하다.
- 고가 장비는 자산 처리 여부를 라벨만으로는 확정할 수 없다.

## H. 다음 단계 연결 포인트
- 규칙 엔진은 최소 아래 상태값을 내려줄 수 있어야 한다.
  - `high_likelihood`
  - `needs_review`
  - `do_not_auto_allow`
  - `consult_tax_review`
- UI는 현재 라벨을 그대로 재사용하고, 규칙 엔진은 `short_reason`, `why_detail`, `guide_anchor`를 함께 내려주면 된다.
- 거래처 식사, 주말/심야, 고가 장비는 추가 질문 흐름으로 연결하는 것이 적절하다.
