# 공식 자료 기능 법적 경계 보고서

## 목적
- 공식 자료 업로드/파싱/저장/상태반영 기능을 구현하기 전에, 현재 단계에서 열 수 있는 범위와 열면 안 되는 범위를 공식 법령과 공식 기관 기능 기준으로 고정한다.
- 이 문서는 기능 확장 문서가 아니라 구현 제한 문서다.

## 공식 근거
- 법제처 국가법령정보센터
  - 개인정보 보호법 제15조(개인정보의 수집·이용)
  - 개인정보 보호법 제16조(개인정보의 수집 제한)
  - 개인정보 보호법 제21조(개인정보의 파기)
  - 개인정보 보호법 제23조(민감정보의 처리 제한)
  - 개인정보 보호법 제24조(고유식별정보의 처리 제한)
  - 개인정보 보호법 제29조(안전조치의무)
  - 개인정보 보호법 제30조(개인정보 처리방침의 수립 및 공개)
  - 전자정부법 시행령 제6조(전자화문서의 진본성 확인)
- 공식 기관 기능
  - 정부24 `다운로드파일 진본확인`
  - 홈택스 `민원증명 원본 확인(수요처조회)`
  - 국민건강보험공단 `증명서 발급사실 확인`

- 법제처 국가법령정보센터 `전자정부법 시행령 제6조`: https://www.law.go.kr/법령/전자정부법시행령/제6조

- 홈택스/손택스 공식 매뉴얼(민원증명 원본 확인 메뉴 포함): https://mob.hometax.go.kr/jsonAction.do?actionId=UTBPPABA40F001
- 국민건강보험공단 증명서 발급사실 확인 안내: https://www.nhis.or.kr/static/html/guide/sub1_04.html
- 국민건강보험 사회보험통합징수포털 제증명 발급/진위확인 안내: https://si4n.nhis.or.kr/jpca/JpCab00105.do

## 판단 기준
- 사용자 업로드 파일은 기본적으로 `사용자 업로드 자료`다.
- 기관의 공식 확인 기능을 거치기 전에는 `기관 확인 완료`로 취급하지 않는다.
- 구조 검증 성공과 기관 진위확인은 같은 의미가 아니다.
- 법적 근거가 불명확하면 구현하지 않는다.

## A. 지금 구현 가능한 범위
- 사용자 업로드 파일에 대해 형식 whitelist 검사 수행
- 문서 종류 식별과 구조 검증 수행
- 기준일, 문서기간, 파싱 상태, 신뢰등급, 해시, 최소 메타데이터 저장
- 세금/건보료 계산에 필요한 최소 핵심 추출값 저장
- stale 여부, 재확인 권장 여부 같은 파생 상태값 저장
- 기관 공식 확인 기능을 사용하는 경우에만 `기관 확인 완료` 메타 저장

## B. 지금 구현 금지 범위
- 사용자 업로드 파일을 자동으로 `진본`, `법적으로 보증`, `100% 정확`으로 표시
- 주민등록번호 전체 저장
- 건강 상세정보 기본 저장
- 불필요한 문서 원문 전체 저장
- NHIS 업로드 파일 전체를 기본값으로 저장
- 외부 자동 조회, 스크래핑, 대리 인증, 비공식 진위확인
- 기관 확인을 거치지 않은 자료를 `기관 확인 완료`로 자동 판정

## C. 유보 범위
- 홈택스 자료 파일 전체의 선택 저장
- 공식 확인 성공 결과의 장기 보관 범위
- NHIS 파일 전체의 선택 저장
- 세무사용 패키지에 기관 확인 결과를 어떤 수준으로 노출할지
- 기관 확인용 reference id, 조회번호, 해시 대조값의 보존기간

## D. 저장 가능 데이터 범위
- 문서 메타데이터
  - source_system
  - document_type
  - parser_version
  - parse_status
  - verified_reference_date
  - trust_grade
  - file_hash
- 핵심 추출값
  - 기간
  - 합계 금액
  - 이미 빠진 세금
  - 납부 확인 금액
- 파생 상태값
  - stale 여부
  - 재확인 권장 여부
  - 반영 가능 여부

## E. 저장 금지 또는 제한 데이터 범위
- 저장 금지
  - 주민등록번호 전체
  - 건강 상세정보
  - 서비스 목적과 무관한 문서 원문 전체
  - 구조 검증 실패 파일의 긴 원문 preview
- 제한 저장
  - 업로드 파일 전체
  - 기관 확인 reference 값
  - 지급처/가입자 식별키 원문

## F. 신뢰등급 기본선
- `A`: 기관 확인 완료
- `B`: 공식 양식 구조 검증 완료
- `C`: 사용자 업로드 자료 기준
- `D`: 사용자 수정 포함 또는 검토 필요

`A`는 정부24/홈택스/NHIS의 공식 확인 기능 성공이 전제다.  
`B`는 parser가 공식 양식 구조를 읽었다는 뜻일 뿐이고, 기관 진위확인과 동일하지 않다.

## G. 다음 실제 구현 티켓에서 바로 써야 할 규칙
- 기본 저장값은 `문서 메타 + 핵심 추출값 + 파생 상태값`이다.
- 업로드 파일 전체 저장은 기본 `off`다.
- NHIS 자료는 파일 전체 저장을 이번 단계에서 열지 않는다.
- `A` 등급은 공식 기관 확인 성공 메타가 있을 때만 부여한다.
- `B/C/D` 등급에서는 UI 문구를 `구조 검증 완료`, `사용자 업로드 자료`, `검토 필요` 수준으로 제한한다.
- `구조 검증 완료`, `기관 확인 완료`, `검토 필요`는 계산 반영 규칙과 분리해 관리한다.

## 다음 티켓 입력용 요약
```text
- 허용: whitelist 업로드, 구조 검증, 기준일/해시/신뢰등급/핵심 추출값 저장
- 금지: 주민등록번호 전체 저장, 건강 상세정보 저장, NHIS 파일 전체 기본 저장, 진본/법적 보증 표현, 자동 스크래핑/대리 인증
- 유보: 홈택스 파일 전체 선택 저장, NHIS 파일 전체 선택 저장, 기관 확인 결과 장기 보관 범위
- 신뢰등급: A=기관 확인 완료, B=공식 양식 구조 검증 완료, C=사용자 업로드 자료 기준, D=사용자 수정 포함 또는 검토 필요
- 구현 기본값: 사용자 업로드 자료는 기관 확인 전까지 A로 올리지 않음
```
