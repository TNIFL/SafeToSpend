# 공식 자료 반영 우선순위 규칙

## 목적
공식 자료 업로드/파싱 결과를 실제 세금/건보료 계산과 신뢰도 UX에 연결할 때, 무엇을 자동 반영하고 무엇을 반영하지 않을지 보수적으로 고정한다.

## 기본 원칙
- `parsed` 상태이고 필수 필드 검증을 통과한 자료만 계산 반영 후보가 된다.
- 공식 자료는 `verified snapshot`으로 취급한다.
- 기준일이 오래된 자료는 자동 반영보다 `재확인 권장`을 우선한다.
- 반영 전/후 값과 반영 출처를 UI에서 설명할 수 있어야 한다.
- NHIS 자료는 v1에서 계산값 강제 덮어쓰기보다 `신뢰도/기준일/참고값` 연결을 우선한다.

## 자료별 반영 대상
### 1. 홈택스 이미 빠진 세금/원천징수 자료
- 반영 가능 항목
  - `이미 빠진 세금`
  - `이미 낸 세금`과 같은 세금 차감 입력의 일부 보정
  - 세금 정확도/신뢰도 보정
- 자동 반영 방식
  - 세금 계산의 차감 입력값으로만 사용한다.
  - 원천징수/기납부 입력이 비어 있거나 더 오래된 경우에만 우선 반영한다.
- 자동 반영 금지 항목
  - 총수입 확정
  - 업무경비 확정
  - 과세유형/업종 추정 확정

### 2. 홈택스 사업용 카드 사용내역
- 반영 가능 항목
  - 증빙/업무지출 참고 정보
  - 비용 추정 신뢰도 보강 메시지
- 자동 반영 금지 항목
  - `자동 비용 확정`
  - `사업용 카드 사용 = 전액 업무경비` 연결
- v1 원칙
  - reference/support 레이어로만 사용한다.

### 3. NHIS 보험료 납부확인서
- 반영 가능 항목
  - 건보료 기준일 보강
  - 납부 확인 상태 표시
  - 참고 납부금액 표시
  - 건보료 신뢰도 보정
- 자동 반영 금지 항목
  - 현재 건보료 계산값 무조건 덮어쓰기
  - 가입유형/소득구조 자동 확정

## 우선순위 규칙
- 기본 우선순위
  - `공식 자료(parsed + fresh + 필수값 정상)` > `수기 입력` > `거래 추정`
- 예외
  - 사용자가 더 최신 시점에 수기 입력을 저장했고, 그 입력이 해당 항목을 명시적으로 포함하면 `수기 입력`을 유지한다.
  - 기준일이 오래된 공식 자료는 `참고/재확인 권장`으로 내리고 자동 반영하지 않는다.

## 기준일 / 유효기간
- 홈택스 세금 자료
  - 기본 120일 이내 자료만 자동 반영 후보
  - 4~6월, 10~11월 시즌에는 60일 이상 경과 시 재확인 우선
- NHIS 자료
  - 기본 90일 이내 자료만 신뢰도 보정 후보
  - 4~6월, 10~11월 시즌에는 45일 이상 경과 시 재확인 우선
- 기간 불일치
  - 문서 기간이 현재 확인 중인 월과 너무 멀면 `stale/period_mismatch`로 내린다.

## NHIS 보수적 처리 원칙
- v1에서는 `공식 자료 기준 확인` 상태를 보여주는 데 집중한다.
- 계산값 자체를 강하게 덮어쓰지 않는다.
- 공식 자료가 있어도 `기준일`, `납부 확인 상태`, `재확인 권장 여부`를 함께 보여준다.

## explainability 원칙
- 어떤 자료가 반영됐는지 문서명/기준일/문서타입으로 설명 가능해야 한다.
- 어떤 자료가 무시됐는지 `ignored_documents`, `stale_documents`로 남겨야 한다.
- `사업용 카드 사용내역 = 자동 비용 확정` 같은 과도한 연결은 금지한다.
