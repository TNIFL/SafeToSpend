# Natural Flow Copy Guide

## 카피 원칙

- 결과 먼저 말한다.
- 세무 용어보다 생활 언어를 먼저 쓴다.
- 왜 묻는지 같이 설명한다.
- `잘 모르겠어요` 또는 `대충 골라도 괜찮아요` 톤을 허용한다.
- 단정 대신 개선 효과를 말한다.

## 핵심 치환표

| 기존 표현 | 우선 표현 | 보조 설명 |
| --- | --- | --- |
| 원천징수 | 돈 받을 때 미리 빠진 세금 | 필요 시 `(원천징수)` 병기 |
| 원천징수 3.3% | 돈 받을 때 3.3%가 떼이는 경우 | 상세 설명에서만 세무 용어 병기 |
| 과세유형 | 부가세를 따로 받는 방식 | 상세 화면에서만 `과세유형` 병기 |
| 필요경비 | 일하면서 쓴 비용 | 안내 페이지에서 필요경비 병기 가능 |
| 업무 경비 | 일하면서 쓴 비용 | 숫자 카드/요약에서 우선 사용 |
| 반영 보류 | 아직 검토가 필요해요 | 상태 설명 우선 표현 |
| 필수 입력 | 먼저 알려주면 좋은 정보 | blocked 상태 등 정말 필요한 경우만 `필수` 사용 |
| 세금 설정 | 기본 정보 이어서 입력 / 정확도 올리기 | nav/깊은 화면은 유지 가능 |

## 화면별 적용 톤

### 온보딩
- `결과부터 보여드릴게요`
- `딱 4개만 고르면 지금 상태 기준 결과를 먼저 보여드려요`
- `대충 골라도 괜찮아요`

### overview
- `결과 더 좋아지게 만들기`
- `이 정보 1개만 더 있으면 예상세금이 더 정확해져요`
- `이 답변을 마치면 영수증이 비용 반영 대상으로 바뀔 수 있어요`

### review
- `세금 정확도 올리기`
- `지금 숫자는 이미 보여드리고 있고, 이 정보만 더 있으면 더 정확해져요`
- `아직 검토가 필요해요`

### tax_buffer
- `지금 보이는 숫자를 더 정확하게 만들기`
- `돈 받을 때 3.3%가 떼이는지와 올해 들어온 돈/일하면서 쓴 비용 정도만 알려주면 돼요`

## 금지 톤

- `반드시 입력해야 합니다`를 전면 문구로 반복
- `복식부기`, `기준경비율`, `과세유형` 같은 용어를 초기 화면에서 설명 없이 바로 제시
- `입력하지 않으면 사용할 수 없습니다` 식의 압박형 문구
