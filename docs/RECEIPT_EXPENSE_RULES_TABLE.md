# 영수증 비용처리 규칙표 초안

기준 상태값:
- `high_likelihood`
- `needs_review`
- `do_not_auto_allow`
- `consult_tax_review`

원칙:
- `high_likelihood`도 자동 인정 확정이 아니다. 설명 가능성이 높다는 뜻으로만 쓴다.
- 애매하면 `needs_review` 또는 `consult_tax_review`로 내린다.

| --- | --- | --- | --- | --- | --- |
| 택시/버스/지하철/주차/통행료 | high_likelihood | 자동 후보만 가능 | 경우에 따라 필요 | 소득세법 제27조 | 비용처리 가능성이 높은 편이에요 |

## 추가 질문 필요 기준

### `needs_review`로 보낼 때 기본 질문
- 누구와, 어떤 목적으로 사용했나요?
- 업무와 직접 관련된 지출인가요?
- 개인 사용과 섞인 부분이 있나요?
- 영수증 외에 거래 사실을 보여줄 자료가 있나요?

### `consult_tax_review`로 보낼 때 기본 질문
- 단순 소모품인가요, 자산 취득인가요?
- 여러 달에 걸쳐 사용하는 장비인가요?
- 일반 업무경비가 아니라 별도 제한 규정 검토가 필요한 지출인가요?

## 보수 분류 메모
- 카테고리가 맞더라도 `merchant_name`, `memo`, `candidate_transaction` 정합성이 약하면 한 단계 낮춘다.
- `weekend_flag`, `late_night_flag`는 단독 차단 사유가 아니라 `needs_review` 가중치로 사용한다.
- 접대비·경조사비·선물은 사용자 메모가 있어도 자동 인정으로 올리지 않는다.
