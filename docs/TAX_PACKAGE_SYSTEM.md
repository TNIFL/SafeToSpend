# 세무사 패키지 시스템 정리

## 목적

세무사 전달 패키지는 월 단위 거래, 증빙, 공식자료, 참고자료를 한 번에 넘기기 위한 ZIP 산출물이다.

- 핵심 목적: 세무사가 `무엇부터 봐야 하는지`, `어디가 재확인 대상인지`, `어떤 자료가 빠졌는지`를 빠르게 파악하게 하는 것
- 기본 원칙: 자동 확정처럼 보이는 표현을 피하고, 요약/상태/재확인 포인트 중심으로 전달
- 기본 정책: 공식자료 원본, 참고자료 원본, 민감 원본은 기본 패키지에 포함하지 않음

## 지원 패키지 종류

현재 지원 profile은 4종이다.

1. 공통형
- 종합 검토용 기본 패키지
- 거래·증빙·원천징수·부가세·건보/연금·참고자료를 함께 본다

2. 종합소득세용
- 종합소득세 검토 보조용
- 원천징수·기납부세액, 공식자료 교차검증, 거래/증빙을 앞쪽에 둔다

3. 부가세용
- 부가세 검토 보조용
- 부가세 요약과 부가세 관련 확인필요항목을 앞쪽에 둔다
- 현재 구현 기준으로 `09_건보_연금_요약.xlsx`는 포함하지 않는다

4. 건보·연금 점검용
- 건강보험/국민연금 자료 존재 여부와 재확인 포인트를 먼저 보는 가벼운 점검 패키지
- 현재 구현 기준으로 `05_원천징수_기납부세액_요약.xlsx`, `08_부가세_자료_요약.xlsx`는 포함하지 않는다
- 국민연금은 현재 구조상 보조 수준일 수 있다

## 공통 workbook 역할

1. `00_패키지요약.xlsx`
- 검토 시작점
- 패키지안내, 반영현황, 공식자료 목록/상태/핵심값, 교차검증 v1 요약 포함

2. `01_사업_상태_요약.xlsx`
- 사용자 유형, 건강보험 상태, 과세 상태
- 값 출처, 확인 수준, 비고 포함

3. `03_거래원장.xlsx`
- 거래 목록
- 분류/신뢰/증빙 연결 상태
- 대표 첨부 상대경로 링크 포함

4. `04_증빙상태표.xlsx`
- 증빙 상태와 거래별 대표 첨부 연결
- 첨부 상대경로 링크 포함

5. `05_원천징수_기납부세액_요약.xlsx`
- 원천징수 이벤트 보조 요약
- 자료 종류, 기준 기간, 총지급액, 원천징수세액, 기납부세액, 지급처 참조, 재확인 필요 포함
- 원천세/지급명세서 이벤트 전용 완성 패키지는 아님

6. `06_세무사_확인필요목록.xlsx`
- 우선확인순서 기준 재확인 목록
- 현재상태, 필요한 확인 내용, 요약설명 중심

7. `07_첨부인덱스.xlsx`
- 패키지에 포함된 첨부 인덱스
- 상대경로 링크 포함

8. `08_부가세_자료_요약.xlsx`
- 부가세 검토 보조 요약
- 과세 상태, 최근 부가세 신고 여부, 세금계산서/카드/현금영수증 관련 합계

9. `09_건보_연금_요약.xlsx`
- 건보·연금 점검 보조 요약
- 건강보험 상태, 건보/연금 자료 존재 여부, 합계, 기준 자료

10. `10_참고자료_요약.xlsx`
- 공식자료 우선, 거래 합계 보조 기준의 참고자료 비교 시트
- 참고자료는 공식자료 대체가 아니라 보조 설명 자료로만 취급

## 민감 원본 기본 제외 원칙

기본 ZIP에는 아래 원본을 포함하지 않는다.

- 공식자료 원본 파일
- 참고자료 원본 파일
- 주민등록번호 전체가 포함될 수 있는 원본
- 건강 상세 원문, NHIS 원본, 가족관계/공제 원본
- 계좌번호/카드번호 전체가 노출될 수 있는 원본

현재 기본 첨부는 `attachments/evidence/` 아래 대표 증빙만 포함한다.

## 현재 지원 범위

- 공통형/종합소득세용/부가세용/건보·연금 점검용 profile 선택
- profile별 ZIP 파일명, 상단 요약, 권장 검토 흐름, 확인필요목록 정렬 차이
- 공식자료 교차검증 v1 결과의 보수적 요약 반영
- 참고자료 보조 연결 상태와 차이 설명
- 상대경로 링크 기반 첨부 접근

## 현재 미지원 / 보류 범위

1. 원천세/지급명세서용 전용 profile
- 현재 `05_원천징수_기납부세액_요약.xlsx`는 이벤트 보조 요약 수준
- 전용 profile로 부르기엔 문서 구분/귀속기간/지급처/확인필요 규칙이 더 필요함

2. 조건부 위험 원본 첨부
- 정책, 동의, 보안 기준이 정해지지 않음

3. 교차검증 결과 직접 반영 2차
- 거래원장 수준의 직접 힌트, 별도 상세 시트 등은 아직 미구현

4. 건보 전용 또는 국민연금 전용 더 작은 패키지
- 현재는 `건보·연금 점검용`까지만 지원

5. 지급명세서/원천세 이벤트 상세 패키지
- 지급처별 상세, 귀속기간별 상세, 제출 단위 구분은 아직 없음

## 다시 열 후보

다음 조건 중 하나가 생기면 세무사 패키지 작업을 다시 여는 것이 맞다.

1. 실제 세무사 피드백에서 시트 과다/정보 부족이 반복됨
2. 원천징수 summary를 더 세분화할 필요가 생김
3. 조건부 위험 원본 첨부 정책이 확정됨
4. 교차검증 결과를 거래/문서 레벨로 더 직접 반영해야 함
5. 이벤트별 파생 패키지를 더 세분화해야 함

## 운영 판단

현재 상태는 “실사용 가능한 공통 마스터 패키지 + 3종 파생 검토 패키지 + 1종 점검 패키지” 수준이다.

- 공통형: 종합 검토 기준
- 종합소득세용: 실제 활용도 높음
- 부가세용: 보조 패키지로 실사용 가능
- 건보·연금 점검용: 가벼운 점검 패키지로 실사용 가능

다만 아래는 아직 과장 없이 보류로 보는 것이 맞다.

- 원천세/지급명세서 전용 패키지
- 자동수집 기반 자동 판정 패키지
- 원본 중심 전달 패키지
