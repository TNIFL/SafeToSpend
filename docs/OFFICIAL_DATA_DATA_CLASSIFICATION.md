# 공식 자료 데이터 분류 기준

## 1. 문서 메타데이터
- 예시
  - source_system
  - document_type
  - parser_version
  - parse_status
  - parse_error_code
  - verified_reference_date
  - trust_grade
  - file_hash
- 저장 가능 여부: 가능
- 기본 저장 여부: 저장
- 사용자 상태 반영 가능 여부: 가능
- 보유기간 방향: 목적 달성, 대체 자료 확보, 탈퇴 시 정리
- 법적 주의사항
  - 해시는 기관 확인 대체 수단이 아니라 업로드 이후 무결성 추적용이다.
  - 파일명은 경로 제거/정규화 전제다.

## 2. 핵심 추출값
- 예시
  - 발급기관
  - 문서명
  - 문서 기간
  - 합계 금액
  - 이미 빠진 세금
  - 납부 확인 금액
- 저장 가능 여부: 가능
- 기본 저장 여부: 저장
- 사용자 상태 반영 가능 여부: 가능
- 보유기간 방향: 최신 스냅샷 유지, 대체 자료 확보 시 이전 자료 정리
- 법적 주의사항
  - 계산/상태 설명에 필요한 최소 필드만 남긴다.
  - 지급처 식별키, 가입자 식별키 같은 값은 기본 비저장 또는 비가역 해시를 우선 검토한다.

## 3. 파생 상태값
- 예시
  - stale 여부
  - 재확인 권장 여부
  - 반영 가능 여부
  - 구조 검증 성공 여부
  - 기관 확인 완료 여부
- 저장 가능 여부: 가능
- 기본 저장 여부: 저장
- 사용자 상태 반영 가능 여부: 가능
- 보유기간 방향: 최신 상태 중심 유지
- 법적 주의사항
  - 상태값은 원문 전체를 대신하는 요약이어야 한다.
  - `기관 확인 완료`와 `구조 검증 완료`를 절대 혼동하지 않는다.

## 4. 업로드 파일 전체
- 예시
  - 사용자가 올린 PDF/CSV/XLSX 파일
- 저장 가능 여부: 제한적
- 기본 저장 여부: 비저장
- 사용자 상태 반영 가능 여부: 불필요
- 보유기간 방향: 별도 동의, 별도 보유기간, 별도 삭제 규칙 없이는 저장 금지
- 법적 주의사항
  - 기본값으로 두지 않는다.
  - NHIS 자료는 현재 단계에서 특히 비저장 원칙을 유지한다.

## 저장 금지 항목
- 주민등록번호 전체
- 건강 상세정보
- 서비스 목적과 무관한 부가 텍스트
- 구조 검증 실패 파일의 긴 원문 preview
- 불필요한 문서 원문 전체

## NHIS 별도 주의
- NHIS 자료는 건강 관련 정보와 고유식별정보가 섞일 가능성이 높다.
- 현재 단계 기본값은 아래다.
  - 파일 전체: 비저장
  - 핵심 추출값: 기준일, 기간, 납부 확인 금액, 최소 상태값만 검토
  - 건강 상세정보: 비대상
  - 주민등록번호 전체: 저장 금지

## 구현 기본값
- 저장 중심축은 `문서 메타 + 핵심 추출값 + 파생 상태값`이다.
- 업로드 파일 전체는 기본 비저장이다.
- 검증 메타와 파일 전체 보관은 분리한다.
