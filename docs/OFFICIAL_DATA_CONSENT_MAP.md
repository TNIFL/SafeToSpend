# 공식 자료 동의 맵 초안

## 동의 구조
### 1) 서비스 운영 기본 데이터
- 목적: 로그인, 월별 분석, 기본 서비스 제공
- 거부 가능 범위: 필수 항목은 거부 시 서비스 핵심 기능 사용이 제한될 수 있음

### 2) 공식 자료 업로드/분석
- 목적: 세금/건보료/증빙 정확도 보정, 세무사 전달 보조
- 자료 종류:
  - 홈택스 현금영수증 지출증빙 내역
  - 홈택스 사업용 카드 사용내역
  - 홈택스 전자세금계산서 내역
  - 홈택스 원천징수·지급명세·납부내역
  - NHIS 보험료 납부확인서
  - NHIS 자격득실확인서
- 기본 저장 단위: 핵심 추출값 우선
- 원본 저장: 기본 비저장 또는 선택 저장 구조 검토
- 거부 가능 범위: 업로드를 거부해도 기본 서비스는 쓸 수 있으나, 정확도 보정 기능은 제한될 수 있음

### 3) 민감/고유식별정보 포함 가능 자료 주의
- 대상 예시:
  - NHIS 자료
  - 주민등록번호 일부가 포함될 수 있는 자료
- 고지 방향:
  - 가능한 최소 범위만 처리
  - 원본은 기본 비저장 우선
  - 민감 가능 자료는 별도 주의문구와 저장 방식 요약을 함께 안내

## 자료별 목적/보유/거부 범위
| 자료 | 목적 | 기본 저장 단위 | 보유기간 방향 | 거부 가능 범위 |
| --- | --- | --- | --- | --- |
| 홈택스 지출증빙/사업용카드/전자세금계산서 | 세금/증빙 정확도 보정 | 추출값 우선 | 최신 기준 유지, 교체 시 정리 | 거부 가능, 정확도 기능 제한 |
| 홈택스 원천징수/지급명세/납부내역 | 이미 빠진 세금 보정 | 추출값 우선 | 최신 기준 유지, 교체 시 정리 | 거부 가능, 세금 정확도 제한 |
| NHIS 보험료 납부확인서/자격득실확인서 | 건보료/가입상태 정확도 보정 | 추출값 우선, 원본 기본 비저장 | 기준일 경과·목적 종료 시 정리 | 거부 가능, 건보료 정확도 제한 |
