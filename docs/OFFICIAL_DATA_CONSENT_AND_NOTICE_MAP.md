# 공식 자료 동의/고지 맵

## 1. 업로드 전 사용자 고지
- 업로드 목적
  - 세금/건보료/증빙 숫자를 더 실제에 가깝게 맞추기 위해 사용
- 저장 대상
  - 문서 메타
  - 핵심 추출값
  - 파생 상태값
- 저장하지 않는 항목
  - 주민등록번호 전체
  - 건강 상세정보
  - 업로드 파일 전체 기본 저장
- 기준일/재확인 개념
  - 자료는 `기준일이 있는 스냅샷`으로 다루고, 새 시즌이나 큰 숫자 변화가 있을 때만 다시 확인
- 신뢰등급 개념
  - 기관 확인 완료와 구조 검증 완료는 다름
- 거부 가능 범위
  - 업로드는 선택 사항
  - 거부해도 기본 기능 사용 가능하나 정확도 보정은 제한될 수 있음

## 2. 업로드 후 사용자 고지
- 현재 반영 수준
  - 기관 확인 완료 / 구조 검증 완료 / 사용자 업로드 자료 기준 / 검토 필요
- 기준일
  - 언제 확인한 자료인지 표시
- 재확인
  - 자료가 오래되거나 시즌이 바뀌면 다시 확인 권장
- 주의
  - 구조 검증만으로 기관 확인 완료가 되는 것은 아님

## 3. 처리방침 본문에 들어가야 할 포인트
- 수집·이용 목적
- 저장 대상과 저장하지 않는 항목
- 파일 전체 기본 비저장 원칙
- 기준일/재확인 구조
- 신뢰등급 개념
- 거부 가능 범위
- 파기 원칙

## 4. NHIS 별도 주의 문구
- NHIS 자료는 민감정보 가능성이 있어 가장 보수적으로 다룸
- 기본값은 핵심 추출값과 상태값 중심
- 업로드 파일 전체 저장은 현재 단계에서 열지 않음

## 금지 표현
- `법적으로 보장`
- `진본임을 보증`
- `100% 정확`

## 필수 표현
- `기관 확인 완료 여부에 따라 검증 수준이 다름`
- `일부 자료는 구조 검증 또는 사용자 업로드 기준으로만 반영될 수 있음`
- `기준일이 있는 스냅샷으로 관리`
