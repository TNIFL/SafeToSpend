# 공식 자료 기능 법적 경계 매트릭스

| 항목 | 현재 단계 판단 | 보수 조건 | 공식 근거 | 메모 |
| --- | --- | --- | --- | --- |
| 사용자 업로드 파일 파싱 | 허용 | 동의 또는 이용자 요청 전제, whitelist 형식만 허용, fail-closed | 개인정보 보호법 제15조, 제16조 | 업로드 자체와 구조 검증은 가능 |
| 핵심 추출값 저장 | 허용 | 목적 적합 최소 항목만 저장, 구조 검증/필수값 확인 통과 시에만 저장 | 개인정보 보호법 제15조, 제16조 | 기간, 기준일, 합계 금액, 세금 보정값 중심 |
| 기준일/신뢰등급 저장 | 허용 | 구조 검증 결과와 기관 확인 결과를 분리 저장 | 개인정보 보호법 제15조, 제16조, 전자정부법 시행령 제6조 | `기관 확인 완료`와 `구조 검증 완료`를 혼용 금지 |
| 파일 전체 선택 저장 | 유보 | 별도 동의, 별도 보유기간, 삭제 정책, 안전조치 확정 전 구현 금지 | 개인정보 보호법 제16조, 제21조, 제29조 | 기본값으로 열지 않음 |
| NHIS 파일 전체 저장 | 금지 | 현재 단계 구현 금지 | 개인정보 보호법 제23조, 제24조, 제16조 | 민감정보/고유식별정보 가능성 때문에 가장 보수적으로 처리 |
| 주민등록번호 전체 처리 | 금지 | 저장/표시/로그 모두 금지 | 개인정보 보호법 제24조 | 필요 최소 마스킹도 별도 검토 전까지 제한 |
| 건강 관련 세부정보 처리 | 금지 | 서비스 목적과 무관한 상세 질병/치료/급여 정보 저장 금지 | 개인정보 보호법 제23조 | NHIS 자료라도 상세 건강정보는 비대상 |
| 기관 확인 결과 저장 | 허용 | 정부24/홈택스/NHIS 공식 확인 기능 성공 메타만 최소 저장 | 전자정부법 시행령 제6조, 정부24 다운로드파일 진본확인, 홈택스 원본 확인, NHIS 발급사실 확인 | 확인 채널/시각/결과코드 수준으로 제한 |
| 세무사 패키지 표시 | 허용 | 신뢰등급과 기준일을 함께 표시, 기관 확인 성공 전 과장 금지 | 개인정보 보호법 제30조 | `구조 검증 완료`, `사용자 업로드 자료 기준` 수준으로만 표기 |
| 자동 조회/스크래핑/대리 인증 | 금지 | 현 단계 구현 금지 | 개인정보 보호법 제15조, 제16조, 제29조 | 법적 근거와 위탁/동의 구조 정리 전 금지 |
| `기관 확인 완료` 자동 판정 | 금지 | 공식 확인 기능 성공 메타 없으면 금지 | 전자정부법 시행령 제6조 | 파일 로고/헤더/해시만으로는 안 됨 |
| CSV/XLSX를 A등급으로 승격 | 금지 | 공식 확인 기능 성공 없는 한 금지 | 전자정부법 시행령 제6조, 홈택스/NHIS/정부24 공식 확인 기능 | CSV/XLSX는 편의용/참고용 또는 구조 검증용 |

## 기준선
- `허용`: 보수적 조건을 만족할 때만 구현 가능
- `금지`: 현재 단계 구현 금지
- `유보`: 추가 법률/정책 검토 전 구현 금지

## 금지 표현
- `진본`
- `원본`
- `100% 정확`
- `법적으로 보증`

위 표현은 신뢰등급/결과 화면/패키지 설명 어디에도 직접 사용하지 않는다.  
대신 `기관 확인 완료`, `공식 양식 구조 검증 완료`, `사용자 업로드 자료 기준`, `검토 필요`를 사용한다.
