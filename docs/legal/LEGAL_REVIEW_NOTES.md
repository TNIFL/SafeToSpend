# LEGAL_REVIEW_NOTES

- 작성일: 2026-03-14
- 목적: 정책 문서/페이지 공개 전 법률 검토 포인트 정리
- 전제: 본 문서는 코드 기준 사실점검 메모이며, 법률 자문을 대체하지 않음

## 1. 법률 검토 필요 문구

- 이용약관의 “환불 없음” 문구
  - 현재 코드/화면 일부에 안내 문구가 있으나, 법령상 예외(청약철회·오결제 등) 정합성 검토 필요
- 책임 제한 조항
  - 포괄적 면책으로 해석되지 않도록 소비자보호 강행규정 충돌 여부 검토 필요
- 해지 조항
  - 상태머신 상 period-end 구조는 있으나 이용자 요청 경로/UI 고지 방식 검토 필요
- 외부 서비스 연동 조항
  - 토스/팝빌/OpenAI 관련 “제공/위탁/국외이전” 법적 분류 최종 확정 필요

## 2. 코드상 미확인/애매한 부분

- 운영사 정보(법인명/대표자/사업자등록번호/주소/대표 연락처) 미확인
- 개인정보보호책임자 정보(성명/연락처) 미확인
- 결제/로그/문의 데이터의 확정 보관기간 정책 미확인(일부 테이블은 기간 규정 없음)
- 이용자 직접 해지 요청을 받는 확정 UI/엔드포인트 미확인
- OpenAI 연동 데이터의 계약상 보관/삭제 조건 및 국외이전 고지 요소 미확인

## 3. 운영 전 최종 확정이 필요한 사항

- 서비스 운영 주체 표시사항(사업자/연락처)
- 환불/해지 정책의 이용자 고지 위치 및 동의 절차
- 개인정보처리방침 시행일/변경 고지 절차
- 문의 채널의 공식 응답 SLA/운영시간
- 외부 연동 장애 시 이용자 안내 문구 및 처리 절차

## 4. 통신판매업/표시사항 확인 포인트

- 사업자 정보 표시(상호/대표자/사업자등록번호/통신판매업 신고 여부)
- 유료서비스 가격/결제주기/자동결제/해지 조건 표시
- 환불·청약철회·분쟁처리 절차 표시
- 전자상거래법/약관법/표시광고법 관련 고지 위치 검토

## 5. 개인정보 수집·제공·위탁 고지 누락 가능성

- 팝빌/토스/OpenAI 연동 시 전송 항목별 고지 구체화 필요
- 제3자 제공 여부(법령상 제공 포함)와 처리위탁 여부 구분 명시 필요
- 국외이전 해당 여부(특히 OpenAI) 및 고지 항목 검토 필요
- 로그/보안 데이터(IP/User-Agent)의 보관 기간·파기 기준 명시 필요

## 6. 환불 없음/면책/책임제한 검토 포인트

- “환불 없음” 문구가 예외사유를 과도하게 배제하지 않는지 점검
- 면책고지 문구가 “전면 책임 배제”로 읽히지 않도록 유지
- 세금/건보 추정치 고지가 오인 가능성을 충분히 낮추는지 점검
- 책임제한 조항에 고의·중과실 및 강행규정 예외를 명시했는지 점검

## 7. 문서/페이지 정합성 체크 결과

- 문서 3종 상호 모순: 현재 기준 중대한 모순 없음
- `LEGAL_FACT_MATRIX` 대비 초안 정합성: 허위 운영자 정보/미구현 로그인 방식은 미기재 처리
- docs와 templates 의미 일치성: 핵심 조항(추정치 성격, 외부 연동, 환불/해지 주의) 동일 방향 유지
- 실제 없는 기능/데이터 항목: 소셜로그인, 임의 제3자 제공, 허위 연락처는 미기재

## 8. 최종 권고

- 현 문서는 “코드 기준 서비스용 초안”으로는 사용 가능
- 실제 공개 전에는 반드시 법률 검토 후 다음 항목 확정 필요
  - 사업자/연락처/개인정보보호책임자 정보
  - 환불/해지 정책 문구의 법령 적합성
  - 외부 연동(토스/팝빌/OpenAI) 고지의 법적 분류와 필수 고지 항목
