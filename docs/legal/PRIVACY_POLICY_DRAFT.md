# 개인정보처리방침 (초안)

SafeToSpend(쓸수있어) 서비스(이하 "서비스")는 이용자의 개인정보를 관련 법령에 따라 처리합니다.  
본 문서는 현재 코드베이스 기준 사실관계에 맞춰 작성된 초안입니다.

## 1. 수집하는 개인정보 항목

### 1-1. 계정 및 인증
- 이메일 주소
- 비밀번호(원문이 아닌 해시값 저장)
- 계정 생성 시각, 플랜/구독 상태 관련 식별 값

### 1-2. 서비스 입력/생성 데이터
- 거래 데이터: 거래일시, 금액, 입출금 구분, 거래처, 메모, 분류 결과
- 세금/건보 추정 입력 데이터: 세금 프로필(JSON), 건보 프로필 및 과거 고지 이력
- 증빙/영수증 데이터:
  - 업로드 파일(원본 파일)
  - 파일 메타데이터(파일키, 원본 파일명, MIME, 용량, SHA-256, 업로드 시각, 보관기한)
  - 영수증 파싱 결과(JSON)
- 고객문의 데이터: 문의 제목/내용, 처리상태, 관리자 답변, 조회 시각

### 1-3. 결제/구독 관련 데이터
- 결제/구독 상태 정보, 결제 시도/이벤트 로그
- 결제수단 등록 관련 식별값(암호화/해시된 키 정보 포함)

### 1-4. 보안/접속 및 토큰 관련 데이터
- 세션 정보, CSRF 토큰
- API 인증용 refresh token 해시, 만료/폐기 시각
- 보안/감사 로그(이벤트, 경로, 메서드, 사용자 식별값, 일부 보조 정보)
- 요청 처리 과정에서의 IP 주소, User-Agent(인증/보안 목적)

## 2. 개인정보 수집 방법

- 회원가입/로그인/설정 화면에서 이용자가 직접 입력
- CSV/계좌연동/파일 업로드(증빙/영수증) 과정에서 이용자가 제출
- 서비스 이용 과정에서 자동 생성(분류 결과, 추정 결과, 처리 로그)
- 외부 연동 API(결제/계좌/영수증 파싱) 호출 결과 수신

## 3. 개인정보 이용 목적

- 이용자 식별 및 인증(로그인, 비밀번호 검증, 세션 유지)
- 거래/증빙 정리 및 리포트 제공
- 세금·건보료 추정 결과 산출 및 상태 안내
- 결제/구독 처리, 요금제 권한 부여 및 상태 관리
- 고객문의 접수/처리 및 답변 제공
- 보안 통제(비정상 요청 제한, 감사 로그, 토큰 폐기/재발급)

## 4. 보유 및 이용 기간

코드에서 확인되는 범위 기준으로 아래와 같이 처리됩니다.

- 계정 및 사용자 데이터: 회원 탈퇴 요청 시 삭제 절차 수행
- 증빙 파일:
  - 기본 보관기한은 업로드 시점 기준 7년(기본 설정)
  - 보관기한 만료 시 정리 작업(purge) 대상
  - 회원 탈퇴 시 물리 파일 삭제를 시도하며, 일부 실패분은 후속 정리 가능
- 문의 데이터: 회원 탈퇴 처리 시 삭제 절차에 포함
- 결제/구독/로그 데이터: 별도 보관기간이 코드에 일괄 상수로 명시되어 있지 않은 항목이 있어 운영 정책 확정 및 법률 검토가 필요

## 5. 파기 절차 및 방법

- 회원 탈퇴 시 사용자 연관 테이블 데이터를 순차 삭제
- 업로드 파일은 파일키 기준으로 물리 삭제를 시도
- 증빙 보관 만료 데이터는 정리 작업으로 파일/메타를 제거
- 기술적/운영상 사유로 즉시 삭제가 어려운 경우, 후속 정리 작업으로 제거

## 6. 제3자 제공에 관한 사항

- 현재 코드 기준으로, 이용자 개인정보를 일반적인 마케팅/판매 목적의 제3자 제공 기능은 확인되지 않았습니다.
- 다만 법령상 의무 이행, 수사기관 요청 등 법적 사유에 따른 제공은 별도 검토가 필요합니다.

## 7. 외부 서비스 이용(처리위탁/외부처리 관련)

서비스 기능 제공을 위해 아래 외부 서비스를 연동합니다.

- 토스페이먼츠: 결제수단 등록, 결제 승인/조회, 웹훅 처리
- 팝빌(EasyFinBank): 계좌 조회/거래 수집
- OpenAI API: 영수증 텍스트/이미지/PDF 파싱

위 연동에서 실제 전송 항목, 국외이전·위탁 고지 방식, 계약상 지위(제공/위탁 구분)는 최종 법률 검토 후 확정이 필요합니다.

## 8. 정보주체 권리행사 방법

이용자는 다음 권리를 행사할 수 있습니다.

- 계정 정보 열람/수정(서비스 화면 내 설정)
- 비밀번호 변경
- 계정 삭제(탈퇴)
- 문의를 통한 처리 요청

현재 확인 가능한 공식 문의 경로는 서비스 내 `/support` 입니다.

## 9. 안전성 확보조치

코드 기준으로 확인되는 주요 조치는 다음과 같습니다.

- 비밀번호 해시 저장
- 세션 쿠키 보안 속성 적용(`HttpOnly`, `SameSite`, 환경별 `Secure`)
- 상태 변경 요청에 대한 CSRF 검증
- API Bearer 토큰 검증 및 refresh token 회전/폐기
- 보안 헤더(CSP, X-Frame-Options 등) 적용
- 민감 정보 로그 필터링 적용

## 10. 자동 수집 장치(쿠키/세션 등)

- 세션 쿠키를 사용하여 로그인 상태를 유지합니다.
- CSRF 보호를 위한 토큰을 사용합니다.
- 별도 광고/행태분석 목적의 쿠키 사용 근거는 현재 코드에서 확인되지 않았습니다.

## 11. 문의처 및 개인정보보호 책임 관련

- 현재 코드에서 확인 가능한 문의 경로: `/support` (인앱 문의)
- 운영사 실명, 사업자등록번호, 주소, 개인정보보호책임자 성명/직통 연락처는 코드만으로 확정할 수 없어 운영 정보 확정 후 반영이 필요합니다.

## 12. 변경 고지

본 방침이 변경되는 경우, 서비스 내 공지 또는 정책 페이지를 통해 안내합니다.
