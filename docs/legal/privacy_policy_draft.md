# 개인정보처리방침 초안

- 상태: 초안
- 기준: 현재 `main` 구현
- 주의: 운영주체 정보, 문의처, 동의 로그, 버전 관리, 외부 서비스 연동의 법적 분류가 아직 확정되지 않았으므로 이 문서는 바로 게시용 확정본이 아니다.

## 1. 총칙

`[운영주체명 입력 필요]`는 `쓸수있어(SafeToSpend)` 서비스 운영을 위해 필요한 범위에서 개인정보를 처리합니다. 본 초안은 현재 구현 기준의 실제 처리 사실을 중심으로 작성되었으며, 실제 운영 전에는 문구와 기능의 일치 여부를 다시 확인해야 합니다.

## 2. 회원가입과 로그인 시 처리하는 항목

### 수집 항목

- 이메일 주소
- 비밀번호 해시
- 계정 생성 시각
- 로그인 세션 유지에 필요한 쿠키 정보

### 처리 목적

- 회원 식별
- 로그인 유지
- 기본 계정 관리

### 보유 및 이용 기간

- 계정 유지 기간 동안 보관
- 현재 웹 화면에는 자동화된 회원 탈퇴 기능이 연결되어 있지 않으므로, 정식 운영 전 탈퇴 및 삭제 절차를 별도로 확정해야 함

## 3. 서비스 이용 중 추가로 처리할 수 있는 항목

### 거래 및 설정 정보

- 수기 입력 또는 import된 거래 정보
- 분류 결과, 메모, 라벨, 규칙
- 사용자 설정 정보와 일부 추천형 안내용 메타 정보

처리 목적

- 거래 정리
- 분류 상태 표시
- 세금 보관함, 리뷰, 패키지 기능 제공

보유 및 이용 기간

- 계정 유지 기간 동안 보관
- 별도 삭제 기능이 연결된 데이터는 사용자가 삭제하거나 운영 정책에 따라 정리될 수 있음

### 증빙자료

- 증빙 파일 원본
- 파일명, 크기, 형식, 해시, 업로드 시각, 보관 만료일 등 메타데이터

처리 목적

- 거래별 증빙 첨부
- 증빙 누락 여부 확인
- 보관함 및 패키지 기능 제공

보유 및 이용 기간

- 관련 거래와 연결된 동안 보관될 수 있음
- 일부 기능에는 파일 삭제 및 보관 만료 처리 로직이 연결되어 있음
- 기능별 보관 방식이 완전히 일치하지 않으므로, 정식 운영 전 기준을 단일화해야 함

### 공식자료

- 업로드한 공식자료 원본 파일
- 파일명, 형식, 크기, 해시
- 문서 종류, 기준일, 파싱 상태, 검토 상태, 추출 요약값

처리 목적

- 공식자료 업로드 및 구조화
- 공식자료 상태 표시
- 일부 비교 기능과 패키지 기능 제공

보유 및 이용 기간

- 현재 구현 기준으로 원본 파일과 추출값이 저장됨
- 현재 웹 화면에는 공식자료 삭제 기능이 연결되어 있지 않음
- 따라서 정식 운영 전 보관기간과 삭제 절차를 별도로 확정해야 함

### 참고자료

- 업로드한 참고자료 원본 파일
- 제목, 메모
- 파일 메타데이터

처리 목적

- 세무사 참고용 자료 보관
- 설명 보조 자료 관리

보유 및 이용 기간

- 현재 구현 기준으로 원본 파일과 메타데이터가 저장됨
- 현재 웹 화면에는 참고자료 삭제 기능이 연결되어 있지 않음
- 따라서 정식 운영 전 보관기간과 삭제 절차를 별도로 확정해야 함

### 계좌연동 정보

- 연동된 은행 코드
- 계좌번호
- 별칭
- 최근 동기화 시각
- 연동으로 수집된 거래 내역

처리 목적

- 계좌연동 거래 수집
- 거래 자동 등록
- 거래 검토 기능 제공

보유 및 이용 기간

- 계정 유지 기간 동안 보관될 수 있음
- 정식 운영 전 연동 해제와 데이터 정리 절차를 더 구체화해야 함

### 영수증 파싱 기능

- 업로드한 영수증 이미지
- 파싱 결과 초안
- 작업 이력과 오류 정보

처리 목적

- 영수증 이미지를 구조화된 거래 초안으로 변환
- 사용자의 확인 후 거래 생성 보조

보유 및 이용 기간

- 현재 구현 기준으로 업로드 이미지와 작업 이력이 저장될 수 있음
- 자동 정리 및 파기 정책은 정식 운영 전 보완이 필요함

## 4. 민감정보 및 주민등록번호에 관한 현재 기준

- 회원가입 단계에서는 민감정보와 주민등록번호를 필수로 수집하지 않습니다.
- 다만 사용자가 업로드한 공식자료, 증빙자료, 참고자료 원본 안에 주민등록번호 등 고위험 정보가 포함될 가능성은 있습니다.
- 현재 구현 기준으로 이러한 원본 파일의 최소화·마스킹·비보관 정책은 완전히 정리되지 않았으므로, 정식 운영 전 별도 보완이 필요합니다.
- 특히 건강보험 자격 관련 자료, 건강보험 납부 자료는 일반 세금 자료보다 더 보수적으로 다뤄야 합니다.

## 5. 개인정보의 제3자 제공

- 현재 코드 기준으로, 이용자의 개인정보를 광고 목적 등으로 제3자에게 상시 제공하는 로직은 확인되지 않았습니다.

## 6. 외부 서비스 연동과 처리위탁 관련 초안 기준

- 계좌연동 기능을 사용할 경우 외부 연동 서비스인 Popbill과 통신이 발생할 수 있습니다.
- 영수증 파싱 기능을 사용할 경우 OpenAI API 호출이 발생할 수 있습니다.
- 다만 이 두 기능에 대한 법적 분류를 `처리위탁`으로 볼지 `제3자 제공`으로 볼지는 실제 계약 구조와 운영 방식 확인이 필요합니다.
- 정식 처리방침에는 운영 확정 후 외부 서비스 명칭, 처리 항목, 목적, 법적 관계를 다시 정리해야 합니다.

## 7. 개인정보의 보유 및 이용 기간

- 법령상 보관 의무가 있거나 분쟁 대응이 필요한 경우를 제외하고, 목적이 달성되거나 불필요해지면 지체 없이 삭제 또는 파기하는 것을 원칙으로 합니다.
- 다만 현재 구현 기준으로 자료군별 자동 삭제 정책이 아직 완전히 정리되지 않은 부분이 있으므로, 정식 운영 전 보유기간 정책을 자료별로 다시 고정해야 합니다.

## 8. 개인정보의 파기 절차 및 방법

### 파기 절차

- 보유 목적이 끝났거나 운영 정책상 더 이상 필요하지 않은 데이터는 별도 대상으로 분류해 파기합니다.

### 파기 방법

- 전자파일: 복구가 어렵도록 삭제
- 데이터베이스 정보: 운영 기준에 따라 삭제 또는 비식별 처리

### 초안 단계 유의

- 현재 일부 자료군은 자동 삭제 UI나 자동 파기 스케줄이 아직 완전히 연결되어 있지 않습니다.
- 따라서 정식 게시 전에는 자료군별 파기 방식과 실행 주체를 더 구체화해야 합니다.

## 9. 이용자의 권리

- 이용자는 자신의 개인정보에 대해 열람, 정정, 삭제, 처리정지 요청을 할 수 있습니다.
- 현재 웹 화면에서 이 권리를 전부 자동으로 행사할 수 있는 것은 아니므로, 정식 운영 전 요청 수단과 처리 절차를 별도로 마련해야 합니다.

## 10. 개인정보의 안전성 확보 조치 개요

- 비밀번호는 해시 형태로 저장합니다.
- 세션 쿠키에 기본 보안 옵션을 적용합니다.
- 업로드 파일은 자료 종류별 경로를 나누어 저장합니다.
- 파일 형식과 크기를 제한합니다.
- 경로 검증으로 단순 경로 이탈 시도를 막습니다.

## 11. 쿠키

- 로그인 상태 유지를 위해 세션 쿠키를 사용할 수 있습니다.
- 현재 확인된 범위에서 광고 추적이나 마케팅 목적의 별도 쿠키 사용은 확인되지 않았습니다.

## 12. 처리방침의 변경

- 처리방침이 변경될 경우 시행일과 주요 변경 내용을 고지해야 합니다.
- 정식 운영 전에는 문서 버전과 동의 이력 저장 구조가 함께 마련되어야 합니다.

## 13. 문의처

- 개인정보 보호 관련 문의처는 `[문의 이메일 입력 필요]`로 확정이 필요합니다.
