# Seasonal UX State Map

## 시즌 구분

- `may_filing_focus`
  - 적용 기간: 4월~6월
  - 안내 맥락: 작년 수입과 비용을 정리하고 5월 신고 준비를 서두르는 시기
- `november_prepayment_focus`
  - 적용 기간: 10월~11월
  - 안내 맥락: 상반기 기준으로 숫자를 미리 점검하고 11월 중간예납 전 흔들리는 항목을 줄이는 시기
- `off_season`
  - 적용 기간: 1~3월, 7~9월, 12월
  - 안내 맥락: 다음 시즌 전에 월별 정리 습관을 만드는 시기

## 사용자 상태 요소

| 상태 요소 | 계산 기준 | 쓰이는 카드 |
| --- | --- | --- |
| 세금 정확도 부족 | 기본 정보, 3.3%, 이미 빠진 세금 입력이 부족한지 | 5월 정확도 카드, 11월 기준 점검 카드, 비시즌 정확도 카드 |

| reinforcement 대기 건수 | follow-up 답변은 있는데 보강 정보는 비어 있는 거래 수 | 5월 영수증 정리 카드, 11월 비용 보강 카드 |
| 세무사 패키지 준비 상태 | preflight `pass/warn/fail` | 5월 패키지 준비 카드, 11월 패키지 준비 카드, 비시즌 패키지 카드 |
| 버퍼 부족 여부 | `buffer_shortage_krw` | 11월 버퍼 점검 카드 |

## 자동 추론 강화 v1에 쓰는 저위험 신호

| 신호 | 의미 | 안전한 이유 |
| --- | --- | --- |
| `receipt_pending_count` | 반영 대기 거래 수 | 이미 계산된 review backlog 수치라 추정이 아니라 관측값이다. |
| `reinforcement_pending_count` | 보강 대기 거래 수 | follow-up 이후 추가 행동이 남았는지 보는 관측값이다. |
| `tax_accuracy_gap` | 세금 정확도 입력 부족 여부 | 입력 누락 여부만 보며 세무 상태를 새로 추정하지 않는다. |
| `package_ready` | 전달 준비 상태 통과 여부 | 패키지 preflight 결과를 그대로 사용한다. |
| `receipt_pending_expense_krw` | 아직 반영 대기 중인 비용 금액 | 이미 계산된 pending bucket을 읽는 값이다. |

## 자동 추론 강화 v1에서 금지하는 신호

| 금지 신호 | 금지 이유 |
| --- | --- |
| 거래 패턴만 보고 3.3% 여부 확정 | 입력하지 않은 세금 상태를 공격적으로 추정하게 된다. |
| 거래 패턴만 보고 면세/과세유형 확정 | 법적/실무적 의미가 커서 설명 불가 추정이 된다. |
| 거래 패턴만 보고 기납부세액 수준 추정 | 실제 입력 없이 세금 수치를 강하게 가정하게 된다. |

## 시즌별 메인 메시지

| 시즌 | 메인 메시지 | 노출 강도 |
| --- | --- | --- |
| 5월 | `이번 시즌은 작년 수입과 비용을 정리하는 달이에요` | strong |
| 11월 | `이번 시즌은 상반기 기준으로 미리 점검하는 시기예요` | strong |
| 비시즌 | `이번 달 리듬에 맞춰 필요한 것만 정리해요` | soft |

## 시즌별 우선 행동 3개

### 5월 신고 시즌

1. 작년 수입과 비용 정리 전에 숫자부터 맞추기
2. 반영 대기 영수증 줄이기
3. 세무사 전달 자료 점검하기

### 11월 중간예납 시즌

1. 상반기 기준으로 숫자 먼저 점검하기
2. 미반영 비용/보강 대기 줄이기
3. 지금 기준 부족한 보관액 확인하기

### 비시즌

1. 이번 달 정리 먼저 하기
2. 기본 정보 미리 맞추기
3. 전달 자료 미리 정리하기

## 화면 연결 정책

| 카드 유형 | 기본 연결 화면 |
| --- | --- |
| 정확도/기준 점검 | `overview` -> `tax_buffer` 또는 `profile step=2` |
| 영수증/보강 | `review` |
| 패키지 준비 | `package` |

## 시즌 종료 후 노출 강도 정책

- `may_filing_focus`, `november_prepayment_focus`
  - overview 상단 강한 체크리스트 노출
  - review/tax_buffer/package에서는 작은 컨텍스트 블록 유지
- `off_season`
  - overview에서는 약한 요약 카드로 축소
  - 개별 화면은 `지금 정리해 두면 다음 시즌이 편해져요` 수준의 약한 문맥만 유지
