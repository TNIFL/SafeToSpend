# 공식 자료 보유기간/파기 정책 초안

## 법적 기준 방향
- 개인정보는 보유기간이 끝나거나 처리 목적이 달성되면 지체 없이 파기한다.
- 안전조치와 접근 통제는 자료 성격에 맞게 더 엄격하게 적용한다.

## 제품 기본안
### 추출값
- 기본 보유 단위: 기준일이 있는 핵심 추출값
- 보유 목적: 세금/건보료 정확도 보정, 증빙 정리, 세무사 전달 보조
- 기본 정책 초안:
  - 사용자가 서비스를 이용하는 동안 최신 기준값으로 유지
  - 더 최신 자료로 교체되면 이전 스냅샷은 목적 범위 안에서만 제한 보관 후 정리
  - 탈퇴 또는 이용 종료 후에는 별도 법적 보관 사유가 없으면 파기

### 원본 파일
- 기본 정책 초안:
  - 기본은 비저장
  - 사용자가 원본 선택 저장을 켠 경우에만 짧은 보관기간을 두는 구조 검토
- 권장 방향:
  - 홈택스 원본: 짧은 확인 기간 후 자동 삭제 구조 검토
  - NHIS 원본: 1차 범위에서는 기본 비저장 우선

## 재확인 시점
- 새 시즌이 시작된 경우
- 예상세금/건보료 숫자가 크게 달라진 경우
- 기준일이 오래된 경우
- 사용자가 직접 다시 맞추고 싶다고 요청한 경우

## 파기 원칙
- 목적 달성 후 지체 없이 파기
- 원본을 저장한 경우 원본 먼저 파기하고, 필요한 추출값만 남길지 별도 정책으로 분리
- 민감/고유식별정보 가능 자료는 파기 우선순위를 높게 둔다.

## 사용자에게 보여줄 핵심 설명
- `기본은 필요한 값만 저장하고, 원본 파일은 항상 보관하지 않아요.`
- `기준일이 지나거나 더 최신 자료가 오면 다시 확인하고, 더 이상 필요 없으면 정리해요.`
