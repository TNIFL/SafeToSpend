# 공식 자료 지원 형식 화이트리스트

## 1. v1에서 받는 공식 자료

| 쉬운 이름 | document_type | 공식 출처 | v1 지원 형식 | 비고 |
| --- | --- | --- | --- | --- |
| 이미 빠진 세금/원천징수 자료 | `hometax_withholding_statement` | 홈택스 | CSV, XLSX, 텍스트 추출 가능한 원본 PDF | 원천징수/납부/지급명세 계열 중 구조가 명확한 파일만 지원 |
| 사업용 카드 사용내역 | `hometax_business_card_usage` | 홈택스 | CSV, XLSX, 텍스트 추출 가능한 원본 PDF | 사업용 카드 사용 요약/내역 구조만 지원 |
| 건보료 납부확인서 | `nhis_payment_confirmation` | NHIS | 텍스트 추출 가능한 원본 PDF | NHIS 민감 가능 정보가 있어 더 보수적으로 처리 |

## 2. 지원 형식 whitelist

- `CSV`
- `XLSX`
- 텍스트 추출 가능한 원본 `PDF`

v1 원칙은 whitelist입니다. 위 형식과 위 문서 종류에 정확히 맞는 공식 원본만 처리합니다.

## 3. 명시적 비지원 형식

아래 형식은 v1에서 자동 반영하지 않습니다.

- 스크린샷 이미지(`PNG`, `JPG`, `JPEG`, `WEBP`)
- 사진 촬영본
- 스캔 PDF
- 암호가 걸린 PDF
- 사용자가 다시 편집한 엑셀/PDF
- 복사/붙여넣기 텍스트 문서
- 제목/헤더가 손상된 파일

## 4. fail-closed 원칙

- 제목, 기관명, 기간, 핵심 금액이 **확실히 읽히는 경우만** `parsed` 처리합니다.
- 일부만 읽히거나 구조가 애매하면 `needs_review` 또는 `unsupported`로 종료합니다.
- 지원 범위 밖 파일은 자동 반영하지 않고, 어떤 공식 사이트 메뉴에서 다시 받아야 하는지 안내합니다.

## 5. fixture 정책

실데이터 없이 구현/테스트하기 위해 아래 원칙으로 fixture를 만듭니다.

- 공식 문서 구조를 참조한 비식별/가상 fixture만 사용
- 실명, 주민등록번호, 실제 납부번호, 실제 사업자번호 사용 금지
- parser가 의존하는 제목/헤더/표/기간/금액 필드만 구조-충실하게 유지
- fixture는 parser 회귀 테스트용이라는 점을 문서와 테스트에서 명시
- 실제 문서를 완전히 재현하지 않고, 구조 검증에 필요한 최소 필드만 유지
