# 공식 자료 신뢰등급 정책

## 목적
- 업로드 파일의 검증 수준을 A/B/C/D로 나누고, 각 등급에서 허용되는 표시와 반영 범위를 고정한다.

## 등급 정의
- `A: 기관 확인 완료`
  - 정부24 다운로드파일 진본확인, 홈택스 민원증명 원본 확인, NHIS 증명서 발급사실 확인 같은 공식 기관 기능으로 확인 성공 메타가 있는 상태
- `B: 공식 양식 구조 검증 완료`
  - parser가 공식 양식의 제목, 발급기관, 필수 헤더, 기준일, 기간, 핵심 금액을 구조적으로 확인한 상태
- `C: 사용자 업로드 자료 기준`
  - 파일은 업로드됐지만 구조 검증이 제한적이거나 계산 반영까지 올리기에는 정보가 부족한 상태
- `D: 사용자 수정 포함 또는 검토 필요`
  - 사용자 수정본, 구조 불일치, 필수값 부족, parser `needs_review/unsupported/failed` 상태

## 등급별 사용 원칙

| 등급 | 의미 | 반영 가능 범위 | 세무사용 표시 수준 | 사용자 표시 문구 방향 |
| --- | --- | --- | --- | --- |
| A | 공식 기관 확인 기능 성공 | 계산 반영, 상태 반영, 패키지 표시 가능 | `기관 확인 완료` | `공식 기관 확인을 마친 자료 기준` |
| B | 공식 양식 구조 검증 성공 | 보수적 계산 반영, 상태 반영 가능 | `공식 양식 구조 검증 완료` | `구조 검증을 마친 자료 기준` |
| C | 사용자 업로드 자료 기준 | 참고값/검토용 상태만 | `사용자 업로드 자료 기준` | `자동 반영 전 확인이 더 필요해요` |
| D | 수정 포함/검토 필요 | 자동 반영 금지 | `검토 필요` | `자동 반영하지 않았어요` |

## 금지 표현
- `진본`
- `원본`
- `100% 정확`
- `법적으로 보증`

## 필수 설명 문구
- 구조 검증 완료는 기관 진위확인과 동일하지 않음
- 해시는 업로드 이후 무결성 추적 도구
- CSV/XLSX는 편의용/참고용 성격

## 보조 원칙
- 해시는 원본 증명 도구가 아니라 업로드 이후 무결성 추적 도구다.
- 구조 검증 완료는 기관 진위확인과 동일하지 않다.
- CSV/XLSX는 기관 확인 완료 등급으로 자동 승격하지 않는다.
- `A` 등급은 공식 기관 확인 기능의 성공 메타가 있을 때만 부여한다.

## UI 적용 규칙
- 계산 엔진
  - A: 보수적 자동 반영 허용
  - B: 보수적 자동 반영 허용
  - C/D: 자동 반영 금지 또는 참고값만
- 세무사용 패키지
  - A/B는 등급과 기준일을 함께 표시
  - C/D는 참고자료/검토필요로 낮춰 표시
- 사용자 화면
  - A/B/C/D 모두 기준일을 함께 표시
  - D는 재확인 유도를 우선한다
