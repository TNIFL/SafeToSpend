# Status Notice Minimum

작성일: 2026-03-11
목적: 장애/점검/지연 시 사용자 공지 최소 구조 확정

## 1) 공지 위치(현재/계획)
- 현재 확정: 앱 내부 flash/안내 문구, 지원 페이지 안내
- 계획 필요: 공용 상태 공지 위치 1개 이상(예: 공지 페이지 또는 footer 링크)

## 2) 필수 공지 유형
1. 결제 지연/실패 다발
2. 계좌 동기화 지연
3. 영수증/증빙 처리 지연
4. 점검/배포로 인한 일시 중단

## 3) 공지 템플릿(최소)
### A. 장애 공지
- 제목: "일부 기능 지연 안내"
- 본문: 영향 범위, 시작 시각, 현재 조치, 다음 업데이트 시각

### B. 결제 실패 안내
- 제목: "결제 처리 실패 안내"
- 본문: 재시도 유도, 반복 시 문의 경로(`/support`), 민감정보 미요청 원칙

### C. 동기화 지연 안내
- 제목: "계좌 동기화 지연 안내"
- 본문: 예상 복구 시점, 수동 재시도 가능 여부, 데이터 유실 여부

## 4) 운영 원칙
- 상태가 확정되지 않은 내용은 추정으로 단정하지 않음
- 민감값(order/payment key raw, querystring raw) 공지에 포함 금지
- 공지 후 사후 보고(원인/재발방지) 간략 게시

## 5) 오픈 판정 관점
- 공지 템플릿은 준비됨
- 실제 공지 게시 위치가 확정되지 않으면 `오픈 전 필수 미완료`
