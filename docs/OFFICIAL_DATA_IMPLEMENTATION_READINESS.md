# 공식 자료 구현 준비 기준

## 이번 단계 구현 대상

| 항목 | 기술적 이유 | 법적 이유 |
| --- | --- | --- |
| 업로드 파일 기본 검사 | 지원 형식/크기/파서 범위를 제어하기 위해 필요 | 최소수집, 안전조치, fail-closed |
| 문서 타입 식별 | parser 라우팅과 unsupported 처리에 필요 | 구조 검증과 기관 확인을 분리하기 위해 필요 |
| 구조 검증 | 기준일/기간/금액을 안전하게 읽기 위해 필요 | 사용자 업로드 자료를 과장 없이 처리하기 위해 필요 |
| 교차 검증 준비 | 향후 공식 확인 성공 메타를 연결하기 위한 준비 | 기관 확인 기능을 비공식 추정과 구분하기 위해 필요 |
| 해시 저장 | 업로드 이후 동일 파일/무결성 추적에 필요 | 진위확인 대체가 아니라 안전한 추적 수단으로 사용 |
| 신뢰등급 저장 | UI/계산/패키지 표시에 필요 | 구조 검증과 기관 확인의 법적 차이를 유지하기 위해 필요 |
| 기준일 저장 | stale 판단과 재확인 권장에 필요 | 목적 적합 범위에서 최신성 설명에 필요 |
| 파생 상태값 저장 | 재확인, 검토 필요, 반영 가능 여부 설명에 필요 | 과장된 자동 반영을 막기 위해 필요 |

## 이번 단계 비대상

| 항목 | 기술적 이유 | 법적 이유 |
| --- | --- | --- |
| 자동 스크래핑 | 인증/세션/구조 변경에 취약 | 법적 근거 없는 자동 조회 금지 |
| 법적 검토 없는 대리 인증 | 위임/권한 구조 미정 | 개인정보 처리 근거 불명확 |
| 건강 상세정보 저장 | 계산 목적에 불필요 | 민감정보 처리 제한 |
| 주민등록번호 전체 저장 | 계산 목적에 불필요 | 고유식별정보 처리 제한 |
| 업로드 파일 전체 기본 저장 | 목적 초과 가능성 큼 | 최소수집, 파기, 안전조치 부담 증가 |
| 비공식 로직으로 `기관 확인 완료` 자동 판정 | 설명 불가능 | 전자정부법 시행령 제6조와 공식 확인 기능 취지에 맞지 않음 |

## 후속 대상

| 항목 | 기술적 이유 | 법적 이유 |
| --- | --- | --- |
| 기관 확인 성공 연계 | A등급 도입에 필요 | 공식 기능 기반 확인 메타 필요 |
| 세무사용 패키지 검증 수준 필터 | 패키지별 표시 분기 필요 | 과장 표시 방지 |
| 파일 전체 선택 저장 UI | 선택 저장/삭제 정책 필요 | 별도 고지와 동의 필요 |
| 자동 삭제/보유기간 처리 | 파기 실행 필요 | 개인정보 보호법 제21조 대응 |

## 개발 시작 전 체크리스트
- `기관 확인 완료`와 `구조 검증 완료`를 별도 필드로 분리했는가
- 주민등록번호 전체/건강 상세정보가 저장 경로에 없는가
- preview 텍스트를 최소화했는가
- 파일 전체 기본 저장이 꺼져 있는가
- trust_grade별 금지 표현이 UI 문구에서 제거됐는가
