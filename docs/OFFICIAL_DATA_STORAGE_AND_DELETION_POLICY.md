# 공식 자료 저장 및 파기 정책

## 기본 원칙
- 목적 제한: 세금/건보료 정확도 보정, 증빙 정리, 상태 설명 범위에서만 처리한다.
- 최소수집: 필요한 최소 메타와 핵심 추출값만 저장한다.
- 분리 저장: 검증 메타, 핵심 추출값, 파생 상태값, 업로드 파일 전체는 분리해서 다룬다.
- 기본 비저장: 업로드 파일 전체는 기본 저장값이 아니다.

## 저장 원칙

| 분류 | 기본 저장 | 조건 | 보유기간 방향 | 비고 |
| --- | --- | --- | --- | --- |
| 문서 메타데이터 | 예 | 파싱/상태 설명에 필요 | 목적 달성 또는 대체 자료 확보 시 정리 | source, type, parser_version, status, hash |
| 핵심 추출값 | 예 | 구조 검증과 필수값 검증 통과 | 최신 스냅샷 중심 유지 | 기준일, 기간, 합계, 세금/납부 확인 금액 |
| 파생 상태값 | 예 | 상태 설명과 반영 규칙에 필요 | 최신 상태 중심 유지 | stale, recheck, trust_grade |
| 업로드 파일 전체 | 아니오 | 별도 고지/선택 저장/삭제정책 확정 후에만 가능 | 별도 짧은 보유기간 필요 | NHIS는 현재 단계 비대상 |

## 저장 금지
- 주민등록번호 전체
- 건강 상세정보
- 서비스 목적과 무관한 문서 원문 전체
- 구조 검증 실패 파일의 긴 preview 텍스트
- 사용자가 붙여넣은 부가 텍스트 원문

## 제한 저장
- 지급처 식별키, 가입자 식별키
  - 기본 비저장
  - 필요하면 비가역 해시 또는 축약값 우선
- 기관 확인 reference 값
  - 최소 범위만 저장
  - 확인 성공 메타와 파일 전체 저장을 분리

## NHIS 별도 원칙
- 업로드 파일 전체: 현재 단계 비저장
- 건강 상세정보: 비대상
- 주민등록번호 전체: 저장 금지
- 최소 저장 후보
  - 기준일
  - 기간
  - 납부 확인 금액
  - 기관 확인 완료 여부
  - trust_grade

## 파기 원칙
- 목적 달성 후 지체 없이 파기한다.
- 더 최신 기준 자료가 확보되면 이전 자료를 정리 대상으로 올린다.
- 계정 탈퇴 시 더 이상 필요한 범위를 제외하고 정리 대상으로 올린다.
- 선택 저장 자료는 별도 보유기간이 끝나면 우선 삭제한다.
- 기관 확인 메타와 파일 전체는 각각 독립적으로 삭제 가능해야 한다.

## 구현 전 체크
- 파일 전체 기본 저장이 열려 있지 않은가
- preview 텍스트를 길게 남기지 않는가
- 주민등록번호 전체/건강 상세정보를 저장하지 않는가
- 대체 자료가 들어왔을 때 이전 자료 정리 경로가 있는가
